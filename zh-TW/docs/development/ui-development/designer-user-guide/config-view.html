配置圖

在“配置圖”中的以下部分，可以配置作用於專案的各種設置： General (一般), Display (顯示器), Default Image Configuration (預設圖像配置), Text Configuration (文字配置), Simulator (模擬器), Build (構建) 和Framework Features (框架特性)。

“配置圖”中的一般設置

General (一般)​

“配置圖”中的一般設置

Application Name (應用名稱)
應用名稱是一個唯讀欄位，顯示創建應用時為應用選擇的名稱。

TouchGFX Board Setup (開發板設置)
該欄位顯示創建應用時使用的TouchGFX開發板設置，如果此資訊未使用，將顯示“N/A”。
如果使用此資訊，將顯示TouchGFX開發板設置的名稱和版本。

Selected Language (選擇的語言)
此下拉清單包含在Texts (文字)中配置的語言，用於選擇專案啟動時應顯示的語言。

Skin (皮膚)
此下拉清單設置兩種內建的風格以支援風格設置的小工具使用，可以選擇“藍色”或“黑色”。 If a widget has been configured with a style from the 'Blue' skin, and the skin is changed to 'Dark', the widget will automatically switch to a corresponding style in the 'Dark' skin.

Startup Screen (啟動畫面)
此下拉清單包含所有已添加到專案的螢幕，用於選擇在專案啟動時要顯示的螢幕。

Display (顯示器)​

在該區域，可以配置顯示器設置。

“配置圖”中的顯示器設置

Dimensions
The size of the display can be set through the W(width) and H(height) properties. 但是，如果已經通過TouchGFX開發板設置配置了尺寸，尺寸配置會被禁用。

Display Orientation (顯示器方向)
顯示器方向可以在橫屏和豎屏之間切換。 這還會影響專案中所用圖像轉換為cpp檔的方式。

Color Depth (色深)
此下拉清單包含可供專案使用的色深。 這些都取決於創建項目時使用的TouchGFX開發板設置。

Default Image Configuration (預設圖像配置)​

在該區域，可以設置專案中的圖像使用的預設設置。 這些設置將作用於專案中的所有圖像，除非它們在圖像視圖中被覆蓋

“配置圖”中的“預設圖像配置”設置

Opaque Image Format (不透明圖像格式)
此下拉清單設置生成出的不透明像素資料的圖像格式。 此下拉清單中的可用圖像格式取決於專案的選定色深。

Non Opaque Image Format (透明圖像格式)
此下拉清單設置應以哪種格式生成透明像素資料的圖像。 此下拉清單中的可用圖像格式取決於專案的選定色深。

Section (區段)
此下拉清單設置圖像資料在目標硬體上的存儲位置。 此下拉清單中的可用分區取決於創建專案時使用的TouchGFX開發板設置。

Extra Section (額外區段)
在使用L8圖像格式時，您可以選擇使用此下拉清單將顏色表存儲在另一個分區。 此下拉清單中的可用分區取決於創建專案時使用的TouchGFX開發板設置。

Dither Algorithm (抖動演算法)
此下拉清單設置圖像使用的抖動演算法。

  • No dither: no dithering is applied to the image. 由於不做任何改變，因此這是性能最高的設置。 However, depending on the image, the quality of the image may also degrade visually at lower color depths.
  • Floyd-Steinberg: diffuses the error to neighboring pixels, resulting in fine-grained dithering but sacrificing sharpness.
  • Jarvis, Judice and Ninke: diffuses the error to pixels one step further away compared to Floyd-Steinberg, resulting in coarser dithering but a sharper image. The slowest of the 3 error-diffusion dithering algorithms.
  • Stucki: based on minimized average error dithering but faster and cleaner.

Alpha Dither (Alpha抖動)
此下拉清單設置是否通過Alpha通道使用抖動演算法。

Layout Rotaion
此下拉清單指定圖像資料生成時的旋轉。 如果螢幕方向發生變化，使用此下拉清單按新方向正確渲染圖像。

Text Configuration (文字配置)​

在該區域，可通過選擇符合專案需求的選項來配置文字渲染和存儲選項。

“配置圖”中的“Text Configuration (文字配置)”設置

Remap texts (重新映射文字)
此選項定義相同轉換是否應重新映射。 Remapping texts will combine identical translations and suffixes across all languages, typographies and alignments, resulting in a reduced footprint.
This option is mutually exclusive with the option 'Binary translation files'

A4
This option defines whether or not the horizontal pixel data of glyphs byte are aligned into an A4 format.
This only affects typographies that are configured as 4bpp

Binary translation files
This option defines whether or not the translations in a project should be moved into binary files that can be loaded at runtime.
This option is mutually exclusive with the option 'Remap texts'

Binary font files (二進位字體檔)
此選項定義是否應將專案中的字體檔移動到可在執行時間載入的二進位檔案中。

Mapped storage format (映射存儲格式)
此選項定義是否應將專案中的字體檔以映射存儲格式輸出。

Unmapped storage format (未映射存儲格式)
此選項定義是否應將專案中的字體檔以未映射存儲格式輸出。

Simulator (模擬器)​

在該區域，可以在運行模擬器時為其添加皮膚。 可以為橫屏和豎屏顯示專案添加皮膚。 The X and Y properties determine the position of the simulator on the skin.

Build (建構)​

在該區域，可以重寫點擊“Run Simulator (運行模擬器)”、“Run Target (運行目標)”和“Generate Code (生成代碼)”時執行的指令。

如需重寫指令，只需在文字方塊中寫入任意指令。 如需在重寫後重置指令，點擊指令名稱旁邊的“Reset (重置)”標籤。

“配置圖”中的建構設置

Generate Assets Command (生成資產指令)

通常設置此指令來生成文字和圖像，在TouchGFX Designer生成代碼後執行。

Post Generate Command (發佈生成指令)
此指令通常用於更新各種專案檔案。 The built-in touchgfx update_project commandline tool supports updating the following project files:

  • Visual Studio (.vcxproj)
  • Keil (.uvprojx)
  • IAR (.ewp & .ipcf)
  • STM32CubeIDE (.project & .cproject)
  • STM32CubeMX (.ioc)

但是，可以在這裡寫入任何需要在生成代碼後執行的指令。

Compile Simulator Command (編譯模擬器指令)
該指令通常通過執行TouchGFX Designer生成的Makefile來執行模擬器的專案編譯。

Run Simulator Command (運行模擬器指令)
該指令執行simulator.exe的啟動。

Post Generate Target Command (發佈生成目標指令)
該指令通常用於更新各種專案檔案，多數為STM32CubeMX (.ioc) 專案檔案。

但是，可以在這裡寫入任何需要在生成代碼後執行的指令。

Compile Target Command (編譯目標指令)
該指令執行目標硬體的專案編譯。

Flash Target Command (快閃記憶體目標指令)
該指令執行專案到目標硬體的快閃記憶體。

在該區域，可以啟用/禁用框架中的特性，特別是紋理映射器小工具支援的圖像格式。 可用於優化目標硬體上專案佔用的程式碼量。
